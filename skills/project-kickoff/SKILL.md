## 目标

把一个模糊的新想法，收敛成一个可执行的起步包。

## 适用场景

- 新建 repo
- 做原型
- 从 0 到 1 搭第一版
- 给一个新方向找最小切入点

## 不适用

- 已有 repo 的继续推进或维护
- 主要目标是阅读别人的项目并提炼经验

## 工作方式

1. 先明确目标、约束、成功标准。
2. 如果涉及技术选型，先加载 `tech-preferences`。
3. 把问题压缩成最小可行形态：先做什么，先不做什么。
4. 优先证明最大的不确定性，而不是展开完整体系设计。
5. 输出一个可直接执行的 kickoff packet。

## Kickoff Packet

至少包含：

- 目标
- 约束
- 最小可行形态
- 第一阶段要证明的点
- 下一步 3 个动作

## 避免

- 在第一版之前引入太多抽象层
- 在没有证据时设计一整套长期架构
- 给用户一长串未过滤的技术选项
